# 通用文件存储服务 - 接入手册

> 版本: 1.11.0 | 更新时间: 2026-04-01

## 概述

本服务提供简单的文件存储 API，支持多项目隔离、数据分析、备份管理，适用于各类项目的数据文件托管。

| 项目 | 说明 |
|------|------|
| 服务地址 | `https://api.tstwg.cn` |
| 管理页面 | `https://api.tstwg.cn/` |
| 文件大小限制 | 单文件最大 50MB |
| 认证方式 | API Key（Header 或 URL 参数） |
| 跨域支持 | 已启用 CORS |

---

## 🔑 API Key 说明

API Key 是访问本服务的凭证，**所有项目共用同一个 API Key**。

**获取方式：**
1. 访问管理页面 `https://api.tstwg.cn/`
2. 查看 `.env` 配置文件中的 `FILE_STORAGE_API_KEY` 值
3. 或联系管理员获取

> 💡 API Key 不是按项目分配的，一个 Key 可以操作所有项目。

**Cookie 支持：**
本服务已配置 CORS 支持 Cookie 凭证传递（`credentials: 'include'`），可以在跨域请求中携带 Cookie。

---

## 📁 项目名称（projectId）说明

`projectId` 是你自己定义的字符串，用于隔离不同项目的文件。

**工作原理：**
- 本服务不需要"创建项目"，直接使用即可
- 第一次上传文件时，系统会自动创建对应的项目目录
- 你的项目名称和本服务的 projectId 是**你自己决定的映射关系**

**举例说明：**

| 你的项目 | 建议的 projectId | 说明 |
|---------|-----------------|------|
| 销售看板系统 | `sales-AAAAA` | 存放销售数据文件 |
| 人事管理系统 | `hr-system` | 存放人事相关文件 |
| 个人测试 | `test` | 临时测试用 |

**命名规则：**
- 仅支持：字母、数字、下划线 `_`、短横线 `-`
- 长度：1-64 字符
- 建议：小写字母 + 短横线

**示例：** 假设你在开发一个"订单管理系统"，你可以：
```javascript
const PROJECT_ID = 'order-system';  // 你自己定义的名称

// 上传订单数据
await fileStorage.upload(file, PROJECT_ID, '订单导出');

// 下载订单数据
const blob = await fileStorage.download(PROJECT_ID, 'orders.xlsx');
```

## 🚀 快速接入

### 第一步：确定你的 projectId

给你的项目起一个英文名称，如 `my-project`、`sales-data`。

### 第二步：获取 API Key

从管理员处获取 API Key（所有项目共用）。

### 第三步：复制代码

将以下代码复制到你的项目中：

```javascript
// fileStorage.js - 文件存储工具类
const FILE_API = 'https://api.tstwg.cn/api';
const API_KEY = '你的API密钥';  // 替换为实际的 Key

export const fileStorage = {
  // 上传文件
  async upload(file, projectId, note = '') {
    const formData = new FormData();
    formData.append('file', file);
    formData.append('keep_name', 'true');
    if (note) formData.append('note', note);
    
    const res = await fetch(`${FILE_API}/upload/${projectId}`, {
      method: 'POST',
      headers: { 'X-API-Key': API_KEY },
      credentials: 'include',
      body: formData
    });
    return res.json();
  },
  
  // 下载文件
  async download(projectId, filename) {
    const res = await fetch(
      `${FILE_API}/download/${projectId}/${encodeURIComponent(filename)}`,
      { 
        headers: { 'X-API-Key': API_KEY },
        credentials: 'include'
      }
    );
    return res.blob();
  },
  
  // 获取下载链接
  getUrl(projectId, filename) {
    return `${FILE_API}/download/${projectId}/${encodeURIComponent(filename)}?api_key=${API_KEY}`;
  },
  
  // 文件列表
  async list(projectId) {
    const res = await fetch(`${FILE_API}/files/${projectId}`, {
      headers: { 'X-API-Key': API_KEY },
      credentials: 'include'
    });
    const data = await res.json();
    return data.files || [];
  },
  
  // 删除文件
  async delete(projectId, filename) {
    const res = await fetch(
      `${FILE_API}/files/${projectId}/${encodeURIComponent(filename)}`,
      { 
        method: 'DELETE', 
        headers: { 'X-API-Key': API_KEY },
        credentials: 'include'
      }
    );
    return res.ok;
  },

  // 批量删除文件
  async batchDelete(projectId, filenames) {
    const res = await fetch(
      `${FILE_API}/files/${projectId}/batch-delete`,
      {
        method: 'POST',
        headers: { 'X-API-Key': API_KEY, 'Content-Type': 'application/json' },
        credentials: 'include',
        body: JSON.stringify({ files: filenames })
      }
    );
    return res.json();
  },

  // 追加记录到 Excel 文件
  async appendExcel(projectId, filename, record) {
    const res = await fetch(`${FILE_API}/append/${projectId}`, {
      method: 'POST',
      headers: { 
        'X-API-Key': API_KEY,
        'Content-Type': 'application/json'
      },
      credentials: 'include',
      body: JSON.stringify({ filename, data: record })
    });
    return res.json();
  }
};
```

### 第四步：开始使用

```javascript
import { fileStorage } from './fileStorage';

// 上传
await fileStorage.upload(file, 'my-project', '数据备份');

// 下载
const blob = await fileStorage.download('my-project', 'data.xlsx');

// 获取链接（用于 <a> 标签或 window.open）
const url = fileStorage.getUrl('my-project', 'data.xlsx');

// 列表
const files = await fileStorage.list('my-project');

// 删除
await fileStorage.delete('my-project', 'old-file.xlsx');

// 批量删除
await fileStorage.batchDelete('my-project', ['a.xlsx', 'b.xlsx']);

// 追加记录到 Excel（自动创建或追加到现有文件）
await fileStorage.appendExcel('my-project', 'records.xlsx', {
  '姓名': '张三',
  '金额': 1000,
  '日期': '2026-04-01'
});
```

## API 认证

所有接口（除健康检查外）都需要认证。

**方式一：HTTP Header（推荐）**
```
X-API-Key: 你的API密钥
```

**方式二：URL 参数**
```
?api_key=你的API密钥
```

**Cookie 支持：**

本服务已配置 CORS 支持跨域 Cookie 传递。如需在请求中携带 Cookie（如用户识别 Cookie），需要：

1. **前端配置**：在 `fetch` 请求中添加 `credentials: 'include'`
   ```javascript
   fetch(url, {
     credentials: 'include',
     headers: { 'X-API-Key': API_KEY }
   })
   ```

2. **生产环境建议**：配置具体的允许域名（修改 `app.py` 中的 CORS origins）
   ```python
   CORS(app, 
        supports_credentials=True,
        origins=['https://your-domain.com'],
        ...)
   ```

**错误响应：**
- `401` - 缺少 API Key
- `403` - API Key 无效

## 上传接口详解

```http
POST /api/upload/:projectId
Content-Type: multipart/form-data
X-API-Key: 你的API密钥
```

**参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `file` | ✅ | 文件内容 |
| `keep_name` | ❌ | 设为 `"true"` 保留原始文件名（含中文） |
| `uploader_name` | ❌ | 上传者姓名（留空自动生成设备标识） |
| `note` | ❌ | 备注说明，最多 200 字符 |

**响应：**
```json
{
  "success": true,
  "filename": "数据.xlsx",
  "project": "my-project",
  "size": 123456,
  "uploader_ip": "113.45.67.89",
  "uploader_name": "Win_a3f2",
  "device": "Win/Chrome",
  "location": {"country": "中国", "region": "北京", "city": "北京"},
  "note": "月度数据"
}
```

> 💡 **上传者名称自动生成**：如果不填写 `uploader_name`，系统会基于持久化用户ID（Cookie）自动生成唯一标识（如 `Win_a3f2`、`iOS_b7c1`），确保同一台电脑换网络后仍能识别为同一用户。

> ⚠️ **重要说明**：如果使用后台上传（如服务端脚本、定时任务等），在请求中包含 `User-Agent` 等浏览器信息，以便系统生成更准确的用户标识。若无Cookie，系统会基于IP和请求头信息自动生成指纹标识。

## 文件统计接口

```http
GET /api/stats/files
X-API-Key: 你的API密钥
```

**响应：**
```json
{
  "stats": {
    "my-project": {
      "day_before_yesterday": 0,
      "yesterday": 5,
      "today": 3,
      "week": 20,
      "month": 45,
      "quarter": 120,
      "year": 365,
      "total": 500,
      "latest_time": "2026-01-29T10:30:00",
      "today_uploaders_count": 2,
      "total_uploaders_count": 15,
      "total_size": 52428800,
      "avg_size": 104857.6,
      "active_uploaders_count": 5,
      "daily_avg": 1.5
    }
  },
  "timestamp": "2026-01-29T10:30:00"
}
```

**字段说明：**
- `day_before_yesterday`：前天的文件数
- `yesterday`：昨天的文件数
- `today`：今日文件数
- `week/week/month/quarter/year`：对应时间维度的文件数量
- `total`：总文件数
- `latest_time`：最新文件上传时间
- `today_uploaders_count`：今日上传用户数
- `total_uploaders_count`：去重后的总用户数
- `total_size`：总文件大小（字节）
- `avg_size`：平均文件大小（字节）
- `active_uploaders_count`：本周活跃用户数
- `daily_avg`：日均上传量（近30天）

## 同名覆盖配置

默认情况下，上传同名文件会自动添加时间戳重命名（如 `data.xlsx` → `data_20260401_143025.xlsx`）。

管理员可以开启同名覆盖模式，直接覆盖已有文件：

```http
POST /api/projects/:projectId/overwrite
Content-Type: application/json
X-Admin-Token: 你的管理员Token
```

**请求体：**
```json
{
  "overwrite": true
}
```

```http
GET /api/projects/:projectId/overwrite
X-Admin-Token: 你的管理员Token
```

**响应：**
```json
{
  "project": "my-project",
  "overwrite": true
}
```

## 管理员权限

部分接口（如同名文件覆盖配置、备份恢复）需要管理员权限。

**获取管理员Token：**
1. 访问管理页面 `https://api.tstwg.cn/`
2. 管理员登录后，在 `.env` 配置文件中查看 `ADMIN_TOKEN` 值
3. 在请求头中添加：`X-Admin-Token: 你的管理员Token`

> ⚠️ **注意**：管理员Token具有最高权限，请妥善保管，不要泄露。

## 常用场景代码

### 场景1：Excel 数据文件托管

```javascript
// 上传 Excel
const input = document.querySelector('input[type="file"]');
input.addEventListener('change', async (e) => {
  const file = e.target.files[0];
  const result = await fileStorage.upload(file, 'report-data', '周报数据');
  console.log('上传成功:', result.filename);
});

// 下载并解析 Excel（需要 xlsx 库）
import * as XLSX from 'xlsx';

async function loadExcelData(filename) {
  const blob = await fileStorage.download('report-data', filename);
  const buffer = await blob.arrayBuffer();
  const workbook = XLSX.read(buffer);
  return XLSX.utils.sheet_to_json(workbook.Sheets[workbook.SheetNames[0]]);
}
```

### 场景2：JSON 配置文件

```javascript
// 保存配置
async function saveConfig(config) {
  const blob = new Blob([JSON.stringify(config, null, 2)], { type: 'application/json' });
  const file = new File([blob], 'config.json');
  await fileStorage.upload(file, 'my-app', '配置更新');
}

// 读取配置
async function loadConfig() {
  const blob = await fileStorage.download('my-app', 'config.json');
  const text = await blob.text();
  return JSON.parse(text);
}
```

## 常见问题

**Q: projectId 需要提前创建吗？**  
A: 不需要。第一次上传文件时会自动创建项目目录，直接用就行。

**Q: 我的项目名称怎么和 projectId 对应？**  
A: 你自己决定。比如你的项目叫"销售系统"，你可以用 `sales-system` 作为 projectId。这个名称只要你自己记住就行。

**Q: 多个项目可以用同一个 API Key 吗？**  
A: 可以。API Key 是服务级别的，不是项目级别的。一个 Key 可以操作所有 projectId。

**Q: 返回 401/403 错误？**  
A: 检查 API Key 是否正确，Header 名称是 `X-API-Key`（注意大小写）。

**Q: 中文文件名乱码？**  
A: 上传时加 `keep_name=true`，下载 URL 用 `encodeURIComponent()` 编码。

**Q: 上传同名文件会怎样？**  
A: 默认添加时间戳避免覆盖（如 `data.xlsx` → `data_20260401_143025.xlsx`），可通过管理员接口配置为直接覆盖。

**Q: 如何安全存储 API Key？**  
A: 使用环境变量，如 `.env` 文件，不要硬编码或提交到 Git。

**Q: 支持哪些文件格式？**  
A: 支持任意文件格式，但追加记录功能仅支持 `.xlsx` 格式。
