# BI数据预处理工具 - 聚合输出功能设计

## 需求概述

在现有功能基础上，新增"聚合输出模式"。开启后，按用户选择的聚合字段进行聚合，每个聚合仅输出一行汇总数据，而非保留所有原始行。

## 字段聚合规则

| 字段 | 聚合方式 | 说明 |
|------|---------|------|
| 审批状态 | 最新值 | 取最大监控日期对应的值 |
| 盈利性判断 | 最新值 | 取最大监控日期对应的值 |
| 日商_预计达成比例 | 平均值 | 聚合内该字段的均值 |
| 外卖日商_预计达成比例 | 平均值 | 聚合内该字段的均值 |
| 到店日商_预计达成比例 | 平均值 | 聚合内该字段的均值 |
| 总营业天数 | 最大值 | 即合计监控天数 |
| 外卖营业天数 | 最大值 | 聚合内该字段的最大值 |
| 总门店数量 | 计数 | 聚合内的门店数量 |
| ★开业日期 | 最新值 | 取最大监控日期对应的值 |
| ★监控日期 | 最新值 | 取聚合内最大的监控日期 |
| ★最新审批状态 | 最新值 | 同审批状态 |
| ★最新盈利判断 | 最新值 | 同盈利性判断 |
| ★50%盈利判断 | 默认 | 盈利天数÷监控天数≥50%返回1，否则0 |

**趋势类字段（盈利趋势、盈利趋势斜率、盈利稳定性、盈利环比变化、数据观测期）：聚合后不再输出**

## 实现方案

### 1. 配置变更

在 `config.json` 的 `aggregation` 中新增：

```json
"output_mode": "detail"
```

- `detail`: 保留原始行（默认）
- `aggregated`: 仅输出聚合汇总行

### 2. UI 变更

在"格式设置"标签页的"聚合设置"区域，新增输出模式选项：

```
输出模式: [详细模式(保留原始行) ▼]
         [聚合模式(每组仅输出一行) ]
```

### 3. 数据处理逻辑变更

**详细模式（默认）：** 保持现有逻辑不变。

**聚合模式：**
1. 处理过程中按聚合字段进行 `groupby`
2. 使用 `agg()` 一次性完成所有字段的聚合计算
3. 输出结果只包含去重后的聚合行，不包含原始行
4. 趋势类字段不参与输出

### 4. 文件修改

仅修改 `process_big_file.py`，不新增文件。

### 5. 聚合函数映射

```python
agg_config = {
    '审批状态': 'last',           # 最新值
    '盈利性判断': 'last',         # 最新值
    '日商_预计达成比例': 'mean',   # 平均值
    '外卖日商_预计达成比例': 'mean', # 平均值
    '到店日商_预计达成比例': 'mean', # 平均值
    '总营业天数': 'max',          # 最大值
    '外卖营业天数': 'max',        # 最大值
    '★开业日期': 'last',         # 最新值
    '★监控日期': 'max',          # 最大值
    '★最新审批状态': 'last',      # 最新值
    '★最新盈利判断': 'last',      # 最新值
    '★50%盈利判断': 自定义函数,    # 基于盈利天数占比计算
    '总门店数量': 'count',        # 计数
}
```
