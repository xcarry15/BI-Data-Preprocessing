# BI数据预处理工具 - 重构设计文档

**日期**: 2026-03-26
**版本**: v2.0

---

## 1. 概述

将现有的 BI 数据预处理工具（单文件 Tkinter 应用）重构为灵活可配置的版本，支持用户自定义列映射、输出字段选择、日期格式等配置项，同时保持对现有数据的默认兼容性。

## 3. 功能模块

### 3.1 文件选择区
- 文件路径输入框（可编辑）
- 浏览按钮：打开文件选择对话框
- 支持拖放文件

### 3.2 列映射配置
- **必需字段**（下拉选择，自动检测）：
  - 门店编码
  - 开业时间
  - 监控日期
  - 审批状态
  - 盈利性判断
- **可选字段组**：
  - 外卖营业天数
  - 日商预计达成比例
  - 外卖日商预计达成比例
  - 到店日商预计达成比例

**智能匹配**：选择文件后自动读取列名，与配置项名称进行模糊匹配（精确匹配 > 包含匹配），自动填充下拉框。

### 3.3 输出字段配置
- Checkbox 列表，默认全部勾选
- 支持"全选"/"全不选"快捷按钮
- 字段：
  - ★最新审批状态
  - ★最新盈利判断
  - ★50%盈利判断
  - ★合计盈利天数
  - ★合计不盈利天数
  - ★盈利天数占比
  - ★合计监控天数
  - ★合计外卖天数
  - ★堂食预计占比
  - ★到店预计占比
  - 检查-审批一致性
  - 检查-盈利一致性

### 3.4 格式设置
- 日期格式：yyyy/MM/dd（默认）、yyyy-MM-dd、MM/dd/yyyy 等
- 编码方式：utf-8-sig（默认）、utf-8、gbk

### 3.5 聚合设置
- 盈利阈值：默认 50%
- 聚合字段：默认"门店编码"

### 3.6 配置持久化
- 保存配置：导出为 JSON 文件
- 加载配置：导入已有配置文件
- 路径：与输入文件同目录的 `.bi_config.json`

### 3.7 处理日志
- 滚动文本区域
- 时间戳前缀
- 清空按钮

## 4. 数据流

```
用户选择文件 → 自动检测列名 → 智能匹配配置 → 用户确认/调整配置
                                      ↓
用户点击"开始处理" → 验证配置完整性 → 读取Excel数据
                                      ↓
                              按配置映射列名
                                      ↓
                              执行数据转换逻辑
                                      ↓
                              根据勾选输出字段过滤
                                      ↓
                              导出CSV
```

## 5. 配置默认值

为保证向后兼容，未修改的配置使用当前硬编码逻辑的默认值：

| 配置项 | 默认值 |
|--------|--------|
| 门店编码列 | 门店编码 |
| 开业时间列 | 开业时间 |
| 监控日期列 | 监控日期 |
| 审批状态列 | 审批状态 |
| 盈利性判断列 | 盈利性判断 |
| 日期格式 | yyyy/MM/dd |
| 盈利阈值 | 50% |
| 所有输出字段 | 勾选 |

## 6. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 缺少必需列 | 弹窗提示缺失列名，标红对应配置项 |
| 文件被占用 | 提示用户关闭文件后重试 |
| 处理中断 | 捕获异常，显示错误日志到日志区 |
| 配置无效 | 保存时验证，加载时校验 JSON 完整性 |

## 7. 技术实现

- **框架**: Tkinter（保持轻量）
- **配置格式**: JSON
- **代码结构**: 类 + 方法分离，配置与逻辑分离
- **线程**: 处理在子线程，主线程负责 UI 更新
