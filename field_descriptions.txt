# BI数据预处理工具 - 输出字段说明
# 格式说明：
# 每条记录以 [字段键名] 开头，后跟三行内容：
# 第1行：字段显示名称
# 第2行：简单说明（普通用户也能看懂）
# 第3行：详细计算公式（专业人士可参考）

[latest_approval]
★最新审批状态
该门店最新一天（按监控日期排序）的审批状态
计算公式：取该门店 max(监控日期) 对应的 审批状态 值

[latest_profit]
★最新盈利判断
该门店最新一天（按监控日期排序）的盈利性判断结果
计算公式：取该门店 max(监控日期) 对应的 盈利性判断 值

[profit_50]
★50%盈利判断
判断该门店盈利天数是否超过总监控天数的50%
计算公式：如果 盈利天数占比 ≥ 50%，返回1；否则返回0

[total_profit_days]
★合计盈利天数
该门店在整个监控期内，盈利性判断=1（盈利）的天数总和
计算公式：sum(盈利性判断 == 1 的记录数)

[total_unprofit_days]
★合计不盈利天数
该门店在整个监控期内，盈利性判断=0（不盈利）的天数总和
计算公式：sum(盈利性判断 == 0 的记录数)

[profit_ratio]
★盈利天数占比
盈利天数占监控总天数的比例，反映门店的整体盈利能力
计算公式：合计盈利天数 ÷ 合计监控天数

[total_monitor_days]
★合计监控天数
该门店在监控期内的总记录天数（每条记录代表一天）
计算公式：count(该门店所有记录数)

[total_delivery_days]
★合计外卖天数
该门店外卖营业天数的最大值（反映外卖业务的持续性）
计算公式：max(外卖营业天数)

[dinein_ratio]
★堂食预计占比
堂食营业额占堂食+外卖总营业额的比例
计算公式：到店日商 ÷ (到店日商 + 外卖日商)
说明：到店日商 = 堂食营业额，外卖日商 = 外卖营业额

[delivery_ratio]
★外卖预计占比
外卖营业额占堂食+外卖总营业额的比例
计算公式：外卖日商 ÷ (到店日商 + 外卖日商)
说明：与堂食预计占比互为补充，之和等于100%

[approval_consistency]
★审批一致性
该门店所有审批状态是否一致
计算公式：
  - 如果所有审批状态都是1 → "全1"（持续盈利，获得持续审批）
  - 如果所有审批状态都是0 → "全0"（持续亏损，审批困难）
  - 如果混合 → "混合"（时好时坏，需要关注）

[profit_consistency]
★盈利一致性
该门店所有盈利判断是否一致
计算公式：
  - 如果所有盈利判断都是1 → "全1"（每天都盈利）
  - 如果所有盈利判断都是0 → "全0"（每天都不盈利）
  - 如果混合 → "混合"（时赢时亏）

[profit_trend]
★盈利趋势
用最近N天数据分析盈利走势是变好还是变差
计算公式：
  - ↑上升 = 盈利越来越多（趋势向好）
  - ↓下降 = 盈利越来越少（趋势变差）
  - →平稳 = 盈利基本持平
判定阈值：slope_threshold = 1/观测天数（动态）
说明：阈值随趋势窗口配置动态调整，窗口内盈利比例从0%到100%的变化才判定为有趋势

[profit_trend_slope]
★盈利趋势斜率
量化盈利变化的剧烈程度，是一个有正负号的数值
计算公式：
  - 正数：盈利天数呈上升趋势（数值越大，上升越快）
  - 负数：盈利天数呈下降趋势（数值越小，下降越快）
  - 0：几乎没有变化
  说明：这是趋势线的斜率值，范围通常在 -1 到 +1 之间
趋势判定阈值：|slope| > 1/观测天数 时才判定为上升或下降

[profit_monthly_change]
★盈利环比变化
把最近N天（趋势窗口）分成前后两半，比较前半段和后半段的盈利变化
计算公式：
  - 显示为百分比，如 +50% 表示后半段比前半段多盈利50%
  - -30% 表示后半段比前半段少盈利30%
  - 0% 表示前后持平
  说明：N = 趋势窗口配置天数（建议7-14天）

[profit_stability]
★盈利稳定性
衡量盈利表现是否忽高忽低，越高越稳定
计算公式：
  - 高稳定(70-100分) = 盈利表现很稳定，每天差不多
  - 中稳定(40-69分) = 盈利表现一般，有时好有时差
  - 低稳定(0-39分) = 盈利表现不稳定，经常波动
  说明：基于趋势窗口内盈利比例的标准差计算得出

[data_observation_days]
★数据观测期
实际用于趋势计算的天数（受限于该门店的总记录数和趋势窗口配置）
计算公式：
  - 观测期 = min(门店总记录数, 趋势窗口配置天数)
  - 最低不少于2天（否则无法计算趋势）
  说明：趋势窗口可在"聚合设置"中配置，建议7-14天
